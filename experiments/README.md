## Experiments

This readme described how to capture separate configs or changes for any experiments with this codebase.  This is to share progress.

Note that this is a fork of a public repo and any code changes are public.

## Use patterns

1. Create a branch.  Branch naming convention: yyyymmdd-decription.
1. Add short description in the Branches section
1. If the experiment includes a video or other artifacts that we want to preserve, create a subdir in `experiments` with the name of the branch and use that dir to store any additional artifacts.
1. Push your changes to this branch.
